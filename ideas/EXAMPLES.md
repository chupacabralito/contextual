# Contextual - Usage Examples

## Example 1: Fixing a Confusing Button

**Traditional approach:**
"Make the save button more prominent"

**With Contextual:**
Click button → "@research[users couldn't find save] @design-system[button prominence] @stakeholders[pm said critical path]"

Agent receives:
- The specific user research showing 3/5 users missed the button
- The relevant design-system pattern for primary actions
- Stakeholder context about this being a conversion blocker

Result: Agent makes informed decision based on actual user needs and business priority

## Example 2: Building a New Feature

**MVP flow while building:**
- Click payment form → "@research[security concerns in checkout]"
- Agent adds trust badges based on specific user concern
- Click shipping section → "@stakeholders[CEO wants familiar flow]"  
- Agent implements familiar pattern user expects

## Example 3: Design System Evolution

Annotating a new component:
"@design-system[button primary] but @research[accessibility audit] and @stakeholders[John color feedback]"

Agent:
- Starts with existing button pattern
- Applies accessibility fixes from audit
- Adjusts color based on stakeholder feedback
- Updates design system with new variant

## Example 4: Strategy Alignment

Starting a new dashboard project:
"@strategy[dashboard goals] @research[common dashboard patterns] @stakeholders[avoid information overload]"

Agent leverages:
- The intended business outcome for the dashboard
- Research on dashboard best practices  
- Prior stakeholder concerns about overload
- Creates better starting point

## Example 5: Stakeholder Alignment

Building controversial feature:
"@stakeholders[legal requirements] @stakeholders[design critique notes] @strategy[q3 goals]"

Agent balances:
- Legal constraints from stakeholder notes
- Design team feedback from review notes
- Strategic objectives from planning docs
- Finds solution that satisfies all constraints

## Example 6: Designing with Taste

Creating a pricing page:
"@research[users want transparency] @taste[stripe pricing clarity] @taste[avoid dark patterns]"

Agent understands:
- Users need clear pricing (research)
- Stripe's pricing page as inspiration (taste)
- Avoid manipulative design patterns
- Creates trustworthy, beautiful pricing display

## Example 7: Maintaining Design Quality

Refining a dashboard:
"@taste[edward-tufte-data-ink] @taste[apple-spatial-hierarchy] - too much visual noise"

Agent applies:
- Data visualization principles from Tufte
- Apple's approach to visual hierarchy
- Reduces clutter while maintaining functionality
- Result feels considered, not just functional
